数据持久化-二进制

二进制

Unity中，数据持久化可以使用二进制格式进行存储。通过将数据编码为二进制形式，可以实现更高效的存储和读取操作。同时也可以防止被修改游戏文件数据

C#提供了一个公共类帮助我们转化

BitConverter.GetBytes() 将各种类型转为二进制字节数组

BitConverter.ToXX(,); 将二进制字节数组转为指定类型

同时C#也有一个专门的编码格式类来帮助我们将字符串和字节数组进行转换

byte[] bytes1 = Encoding.UTF8.GetBytes("Heart"); 将字符串以指定编码格式转字节数组

string s = Encoding.UTF8.GetString(bytes1); 将字节数组串以指定编码格式转字符串

文件流

在C#中，文件流是进行文件读写操作的一种常用方式。可以使用FileStream类来创建和操作文件流。

以下是文件流的常用方法：

  • FileStream(string path, FileMode mode)：使用指定的路径和文件打开模式创建一个新的FileStream对象。
  • Read(byte[] buffer, int offset, int count)：从文件流中读取指定数量的字节到缓冲区。
  • Write(byte[] buffer, int offset, int count)：将指定数量的字节从缓冲区写入文件流。
  • Flush 将字节写入文件 写入后 一定执行一次
  • Seek(long offset, SeekOrigin origin)：将文件流中的位置设置为指定的偏移量和起始点。
  • Close()：关闭文件流，并释放与之关联的所有资源。
  • Dispose 缓存资源销毁回收

文件夹操作

C#中，可以使用Directory类进行文件夹的操作。以下是一些常用的文件夹操作方法：

  • Directory.CreateDirectory(path)：创建一个新的文件夹。
  • Directory.Exists(path)：检查指定路径的文件夹是否存在。
  • Directory.Delete(path)：删除指定路径的文件夹。
  • Directory.GetFiles(path)：获取指定路径下的所有文件的路径。
  • Directory.GetDirectories(path)：获取指定路径下的所有子文件夹的路径。
  • Directory.Move(source, destination)：移动文件夹到指定路径。
  • Directory.GetFiles(path, searchPattern)：获取指定路径下符合搜索模式的文件的路径。
  • Directory.GetDirectories(path, searchPattern)：获取指定路径下符合搜索模式的子文件夹的路径。

FileMode 枚举定义了几个文件打开模式的类型

  1. CreateNew：创建一个新文件。如果文件已存在，则引发异常。
  2. Create：创建一个新文件。如果文件已存在，则覆盖该文件。
  3. Open：打开现有文件。如果文件不存在，则引发异常。
  4. OpenOrCreate：打开现有文件，如果文件不存在则创建新文件。
  5. Truncate：打开现有文件并截断文件的长度为零。如果文件不存在，则引发异常。
  6. Append：打开现有文件并在文件末尾追加数据。如果文件不存在，则创建新文件

FileAccess 枚举类型用于指定对文件的访问权限，包括：

  • Read：允许读取文件的权限。
  • Write：允许写入文件的权限。
  • ReadWrite：同时允许读取和写入文件的权限。

FileShare 枚举类型用于指定其他进程是否可以访问已打开的文件，包括：

  • None：不允许其他进程访问已打开的文件。
  • Read：允许其他进程以读取方式访问已打开的文件。
  • Write：允许其他进程以写入方式访问已打开的文件。
  • ReadWrite：允许其他进程以读取和写入方式访问已打开的文件。
  • Delete：允许其他进程删除已打开的文件。
